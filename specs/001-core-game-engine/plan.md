# Implementation Plan: Core Game Engine

**Branch**: `001-core-game-engine` | **Date**: 2026-04-12 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/001-core-game-engine/spec.md`

## Summary

Build a pure TypeScript game engine for Joker 51 that implements all game rules defined in
`joker51_game_rules.md`. The engine exposes a functional, immutable API: every action accepts
a `GameState` and returns a new `GameState` wrapped in an `ActionResult`. No React, no Expo,
no UI imports ‚Äî this module is consumed by the state layer (Phase 5: store) via a clean API
contract defined in `contracts/engine-api.md`.

## Technical Context

**Language/Version**: TypeScript 5.9 (strict mode, already in project)
**Primary Dependencies**: None (engine is pure TS). Dev: Jest + ts-jest (to be added)
**Storage**: N/A ‚Äî engine is stateless; caller persists `GameState`
**Testing**: Jest with ts-jest transformer (not yet in `package.json` ‚Äî must be added)
**Target Platform**: iOS 15+ and Android via Expo (engine itself is platform-agnostic)
**Project Type**: Internal library module ‚Äî consumed by the app's state layer
**Performance Goals**: Round simulation < 50ms; combination validation < 5ms
**Constraints**: Zero imports from React, Expo, or UI layers; strict immutability
**Scale/Scope**: 2‚Äì8 players, 1‚Äì3 decks, up to 12 rounds per game
