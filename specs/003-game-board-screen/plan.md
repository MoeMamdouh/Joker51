# Implementation Plan: Game Board Screen

**Branch**: `003-game-board-screen` | **Date**: 2026-04-12 | **Spec**: [spec.md](spec.md)

## Summary

Implement the interactive game board screen for Joker 51 — a pass-and-play card game board where the active player draws, melds, lays off, and discards cards using the existing engine (Phase 1). The board reads `GameState` from `gameStore`, dispatches actions via a `useGameActions` hook that calls engine functions and persists state to AsyncStorage after each action, and renders a suite of new `src/components/game/` components driven entirely by design tokens.

## Technical Context

**Language/Version**: TypeScript 5.x strict mode (already in project)
**Primary Dependencies**: React Native + Expo SDK ~54, Expo Router, Zustand, i18next + react-i18next, React Native Reanimated 3+, @react-native-async-storage/async-storage
**Storage**: AsyncStorage — key `@joker51/savedSession` (established in Phase 2)
**Testing**: Jest + @testing-library/react-native (jest-expo preset, added in Phase 2)
**Target Platform**: iOS + Android (React Native / Expo)
**Project Type**: Mobile app — single-device pass-and-play card game
**Performance Goals**: Board re-renders ≤ 300ms after each action; round summary ≤ 500ms; 60fps card selection animation
**Constraints**: Offline-only; RTL support for Arabic; no new navigation libraries
**Scale/Scope**: 2–8 players, 1–35 rounds per game; single screen (`/game` route)
